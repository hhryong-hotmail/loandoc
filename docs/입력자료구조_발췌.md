# 입력 자료 구조 발췌

## 1. 회원가입 입력 자료 구조

### 1.1 RegisterRequest 구조체

RegisterRequest {
    id, String, 사용자 아이디 (필수, 공백 불가)
    password, String, 비밀번호 (필수, 최소 8자 이상)
}

---

## 2. 외국인 근로자 정보 입력 자료 구조

### 2.1 ForeignWorkerMasterRequest 구조체

ForeignWorkerMasterRequest {
    user_id, String, 사용자 ID (필수, 기본키)
    name, String, 이름 (필수)
    nationality, String, 국적 (선택)
    passport_number, String, 여권번호 (필수, UNIQUE 제약조건)
    birth_date, String, 생년월일 (필수, YYYY-MM-DD 형식)
    entry_date, String, 입국일자 (필수, YYYY-MM-DD 형식)
    phone_number, String, 전화번호 (필수)
    current_company, String, 현재 회사 (필수)
    email, String, 이메일 (선택)
}

---

## 3. 신용대출 입력 자료 구조

### 3.1 LoanEstimateRequest 구조체

LoanEstimateRequest {
    loginId, String, 로그인 ID (필수)
    nationality, String, 국적 (필수, 예: Vietnam, Thailand, Philippines, China, India 등)
    remainMonths, Integer, 잔여 체류기간 (개월) (필수)
    annualIncome, Double, 연소득 (만원 단위) (필수)
    age, Integer, 나이 (필수)
    workingMonths, Integer, 재직기간 (개월) (필수)
    visaType, String, 비자 종류 (필수, 예: E-9, E-7, E-4, F-2, F-4, F-5, F-6 등)
    healthInsurance, String, 의료보험 유형 (선택, 예: "지역", "직장" 등, KB저축은행의 경우 "지역"은 불가)
}

**비고:**
- 신용대출은 담보 없이 개인의 신용도와 소득을 기반으로 대출 한도를 산정
- 은행별로 비자 종류, 국적, 나이, 재직기간, 연소득 등에 대한 조건이 상이함
- 예상 한도 계산식: 연소득 × 잔여체류기간 × 가중치 / 10 (최고한도 이내)

---

## 4. 담보대출 입력 자료 구조

### 4.1 LoanDashboardRequest 구조체

LoanDashboardRequest {
    req_login, String, 요청자 로그인 ID (필수)
    counseler, String, 상담자 (선택)
    name, String, 요청자 이름 (필수)
    phone_number, String, 전화번호 (필수)
    nationality, String, 국적 (선택)
    req_type, String, 요청 유형 (선택, 예: "신용대출", "담보대출" 등)
    title, String, 제목 (필수)
    content, String, 내용 (선택)
}

**비고:**
- 담보대출은 담보물을 제공하여 대출을 받는 상품
- 대출 상담 요청 게시판 형태로 관리되며, 상담사가 후속 조치를 진행
- 요청 시점은 `created_at` 필드에 자동 기록됨

---

## 5. 대출 신청 입력 자료 구조

### 5.1 LoanApplicationRequest 구조체

LoanApplicationRequest {
    req-yn, String, 대출 신청 여부 (필수, "yes" 값일 경우 대출 신청 처리)
    login-id, String, 로그인 ID (필수)
}

**비고:**
- 대출 신청 시 `foreign_worker_master` 테이블의 `req_date` 필드가 현재 일시로 업데이트됨
- 대출 신청 시점을 기록하여 추후 관리에 활용

---

## 참고사항

1. 모든 날짜 필드는 YYYY-MM-DD 형식을 따름
2. 필수 필드가 누락된 경우 400 Bad Request 응답
3. 중복된 데이터(여권번호, 사용자 ID 등)인 경우 409 Conflict 응답
4. 데이터베이스 오류 발생 시 500 Internal Server Error 응답
5. 모든 문자열 필드는 UTF-8 인코딩을 사용
