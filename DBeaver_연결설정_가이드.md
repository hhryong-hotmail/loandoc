# DBeaver PostgreSQL 연결 설정 가이드

## 연결 정보
- **호스트**: localhost
- **포트**: 5432
- **데이터베이스**: loandoc
- **사용자**: postgres
- **비밀번호**: postgres

## DBeaver 연결 설정 방법

### 1. 새 데이터베이스 연결 생성
1. DBeaver 실행
2. 상단 메뉴: **데이터베이스** → **새 데이터베이스 연결** (또는 `Ctrl+Shift+N`)
3. 데이터베이스 선택: **PostgreSQL** 선택 → **다음**

### 2. 연결 설정 입력
**Main 탭:**
- **호스트**: `localhost`
- **포트**: `5432`
- **데이터베이스**: `loandoc`
- **사용자 이름**: `postgres`
- **비밀번호**: `postgres`
- **비밀번호 저장** 체크

**Driver properties 탭 (선택사항):**
- 필요시 추가 설정

### 3. 연결 테스트
- **테스트 연결** 버튼 클릭
- "연결 성공" 메시지 확인
- **완료** 클릭

### 4. 연결 활성화
1. 좌측 **데이터베이스 탐색기**에서 생성한 연결을 찾습니다
2. 연결을 **우클릭** → **연결** (또는 더블클릭)
3. 연결이 활성화되면 연결 아이콘이 녹색으로 변경됩니다

### 5. 쿼리 실행
1. 연결을 우클릭 → **SQL 편집기** → **새 SQL 스크립트**
2. 또는 상단 메뉴: **SQL 편집기** → **새 SQL 스크립트**
3. 쿼리 입력:
   ```sql
   SELECT * FROM bank_info;
   ```
4. `Ctrl+Enter` 또는 실행 버튼 클릭

## 문제 해결

### "No active connection" 오류가 발생하는 경우:

1. **연결이 비활성화된 경우**
   - 좌측 탐색기에서 연결을 더블클릭하여 활성화
   - 연결 아이콘이 녹색인지 확인

2. **SQL 편집기에서 연결 선택**
   - SQL 편집기 상단의 연결 드롭다운에서 올바른 연결 선택
   - 또는 SQL 편집기에서 우클릭 → **연결 변경**

3. **연결 정보 확인**
   - 연결을 우클릭 → **편집**
   - 연결 정보가 올바른지 확인
   - **테스트 연결** 버튼으로 연결 테스트

4. **PostgreSQL 드라이버 확인**
   - 연결 편집 → **드라이버** 탭
   - 드라이버가 설치되어 있는지 확인
   - 없으면 DBeaver가 자동으로 다운로드

5. **방화벽 확인**
   - Windows 방화벽에서 포트 5432가 허용되어 있는지 확인
